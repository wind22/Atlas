# 一、设计理念与目标

+-----------------------------------------------------------------------+
| **一句话定位**                                                        |
|                                                                       |
| 这不是一个「告诉你什么时候买卖」的择时器                              |
| ，而是一个每天回答「现在市场偏危险还是偏机会」的仪表盘。它把趋势跟踪  |
| 的规则量化成几个分数和一盏红黄绿的灯，帮你判断该进攻、观望还是防御。  |
+=======================================================================+
+-----------------------------------------------------------------------+

## 1.1 系统要回答的三个问题

-   **① 我在哪：**市场现在处于什么状态？------上涨趋势 / 震荡 /
    下跌，用综合趋势分量化。

-   **②
    危不危险：**风险在升高还是降低？------波动、广度、回撤共同决定风险分，越高越该收敛。

-   **③
    机会在哪：**机会出现在哪里？------哪些行业/资产开始走强，用相对强度与动量捕捉。

## 1.2 设计原则

-   **不预测：**系统只识别「已经发生」的趋势与风险，不预测顶底、不猜点位。

-   **多指标共振：**任何单一指标都会失效，只有多个维度共振时才给出强信号，降低假信号。

-   **概率化提示：**输出是状态与概率（分数、制度、灯色），而非「买/卖」的二元指令。

-   **防过拟合：**参数一律采用行业公认值（200日、50日、ADX 25、VIX
    20），不针对历史最优化。

-   **透明可解释：**每一条提示都能追溯到具体规则，可解释、可复盘。

# 三、信号体系：五个维度与评分

系统对每个标的计算五个维度的分数，再合成为「综合趋势分
T」（0--100，越高越强）和「综合风险分
R」（0--100，越高越危险）。这是整个系统的核心。

  ------------------------------------------------------------------------
  **维度**                **回答的问题**           **权重（计入 T）**
  ----------------------- ------------------------ -----------------------
  ① 趋势方向 Direction    价格在往上还是往下       40 分

  ② 动量 Momentum         涨势有多强、是否领先     30 分

  ③ 趋势强度 Strength     这是真趋势还是震荡       15 分

  ④ 广度                  上涨是普涨还是少数拉动   15 分
  Breadth（仅大盘）                                

  ⑤ 风险 / 波动 Risk      下行风险有多大（→ 单独算 计入风险分 R
                          R）                      
  ------------------------------------------------------------------------

## 3.1 趋势方向分 Direction（0--40）

  -----------------------------------------------------------------------
  **判定条件**                        **得分**
  ----------------------------------- -----------------------------------
  收盘价 \> 200 日均线（长期牛熊线）  16

  收盘价 \> 50 日均线（中期）         8

  50 日均线 \> 200 日均线（多头排列） 8

  200 日均线斜率向上（今值 \> 20      8
  日前）                              
  -----------------------------------------------------------------------

## 3.2 动量分 Momentum（0--30）

  ---------------------------------------------------------------------------
  **判定条件**                            **得分**
  --------------------------------------- -----------------------------------
  12 个月动量为正（剔除最近 1             10
  月，标准动量口径）                      

  6 个月收益为正                          6

  相对 SPY 的 3                           8
  个月相对强度为正（个股/行业轮动核心）   

  距 52 周高点 ≤ 5%（贴近新高=强势）      6
  ---------------------------------------------------------------------------

## 3.3 趋势强度分 Strength（0--15）

  -----------------------------------------------------------------------
  **判定条件**                        **得分**
  ----------------------------------- -----------------------------------
  ADX ≥ 25（有明确趋势，而非震荡）    8

  MACD 柱状为正（DIF \>               7
  DEA，动量向上）                     
  -----------------------------------------------------------------------

## 3.4 广度分 Breadth（0--15，仅大盘层）

按「在 200 日线上方的行业 ETF 占比」线性给分：60% 以上给满分 15，40%
给约 7.5，20% 以下趋近
0。广度衡量上涨是否健康------只有少数龙头拉动、多数板块转弱，是典型的顶部内部背离。个股与行业标的的该项默认继承大盘广度。

## 3.5 风险分 Risk（0--100，越高越危险）

风险分独立计算，用于「刹车」。任一危险条件成立即累加分值，上限 100：

  ----------------------------------------------------------------------------------------
  **风险条件**                                         **加分**
  ---------------------------------------------------- -----------------------------------
  收盘价跌破 200 日均线                                +25

  死叉：50 日均线 \< 200 日均线                        +15

  自近 60 日高点回撤 \> 10%（\> 20% 再 +10）           +15

  近 20 日波动率 \> 过去一年均值 × 1.5（波动骤放）     +20

  VIX \> 20（\> 30 再                                  +10
  +10）------大盘/多资产层专用，个股用自身波动率替代   

  广度：\< 40% 行业在 200 日线上方                     +15
  ----------------------------------------------------------------------------------------

+-----------------------------------------------------------------------+
| **综合分怎么算**                                                      |
|                                                                       |
| 综合趋势分 T = 方向(40) + 动量(30) + 强度(15) + 广度(15)，范围        |
| 0--100。综合风险分 R = 上述风险条件累加，范围                         |
| 0--100。两个分数一升一降，共同决定下一节的「市场制度」。              |
+=======================================================================+
+-----------------------------------------------------------------------+

# 四、综合评分与市场制度（Regime）

把 T 和 R
映射到四种「市场制度」，每种对应明确的应对姿态。这是系统给用户的最终结论，用一盏灯表示。

  ----------------------------------------------------------------------------------------------------------------------------
  **制度**          **灯**            **判定条件（按优先级）**                          **应对姿态**
  ----------------- ----------------- ------------------------------------------------- --------------------------------------
  进攻区 Risk-On    🟢                T ≥ 60 且 R ≤ 30                                  顺势持有，可关注强势行业与创新高个股

  警戒区 Caution    🟡                不满足进攻、也未触发防御（如 T 在 35--60，或 T    减少新增仓位，收紧止损，只留最强标的
                                      高但 R 在 30--60 / 广度转弱 / 动量背离）          

  防御区 Risk-Off   🔴                R ≥ 60 或 T ≤ 35（跌破 200 线+死叉+广度崩塌等）   降低风险敞口，转向现金/债券/黄金

  超卖观察 Oversold 🟠                处于防御条件下、但出现企稳信号（重回短均线、RSI   不急于抄底，观察趋势能否重建
                                      从 \< 30 回升）                                   
  ----------------------------------------------------------------------------------------------------------------------------

## 4.1 制度切换的处理

-   **确认机制：**每日记录制度状态，只有当条件持续满足（如连续 2
    日）才确认切换，避免单日噪音反复横跳（whipsaw）。

-   **切换高亮：**制度发生切换（如 🟢→🟡 或
    🟡→🔴）当天，在仪表盘顶部高亮提示，并与昨日对比说明「因为什么变了」。

-   **层级优先：**大盘制度是「总闸」------大盘在防御区时，个股即使评分高，也建议整体降低敞口。

# 五、提示触发规则（Alerts）

除了每日制度灯，系统在关键事件发生时给出离散提示，分「风险」与「机会」两类。

## 5.1 风险类提示

  ------------------------------------------------------------------------
  **触发事件**                        **含义**
  ----------------------------------- ------------------------------------
  大盘首次跌破 200 日均线（由上转下） 制度可能转向防御，最重要的风险信号

  出现死叉（50 日下穿 200 日）        中期趋势转弱确认

  VIX 单日 \> 30 或跳升 \> 50%        市场恐慌，波动风险骤升

  广度骤降（200 线上方行业占比一周内  内部转弱，普涨变分化
  \> 60% 跌至 \< 40%）                

  个股放量跌破关键均线或前低          个股层面的趋势破位

  波动率突破近一年上沿                不确定性上升，仓位应收敛
  ------------------------------------------------------------------------

## 5.2 机会类提示

  -----------------------------------------------------------------------
  **触发事件**                        **含义**
  ----------------------------------- -----------------------------------
  大盘重新站上 200 日均线（由下转上） 制度可能转向进攻

  出现金叉（50 日上穿 200 日）        中期趋势转多确认

  行业相对强度排名跃入前 3            新的领涨板块出现

  个股创 52 周新高 + 成交量 \> 20     放量突破，趋势启动
  日均量 × 1.5                        

  深度回撤后企稳（重回 200 线 + RSI   超卖修复，趋势可能重建
  从 \< 30 回升）                     

  多资产切换 Risk-On（股↑ 债↓ 金↓     市场风险偏好回升
  同步）

# 六、仪表盘设计

仪表盘是一页自上而下的 HTML，从「总结论」到「细节」逐层展开，30
秒内能看懂今天的市场姿态。布局线框如下：

  -----------------------------------------------------------------------
  **仪表盘布局（自上而下）**
  -----------------------------------------------------------------------
  【顶部 · 市场制度灯】 🟢 进攻 / 🟡 警戒 / 🔴 防御 ＋
  一句话解读（如「大盘维持多头，波动温和，风险偏低」）＋ 与昨日的变化

  【大盘卡片区】 SPY \| QQQ \| IWM \| DIA ------ 每张卡显示：综合趋势分
  T、风险分 R、状态标签、距 200 线的位置

  【行业热力图】 11 个行业 ETF
  按相对强度/趋势状态着色（绿=强势领先，红=弱势破位），一眼看清钱往哪流

  【多资产条】 股 / 债 / 金 / 商品 / 美元 ------
  各显示趋势方向箭头，判断整体 Risk-On 还是 Risk-Off

  【自选个股表】 每只显示：趋势状态、动量、距 52
  周高点、风险旗标（🔴/🟡/🟢）、当日触发的提示

  【今日提示区】 当天触发的所有风险/机会 Alert 列表，按重要性排序
  -----------------------------------------------------------------------

配色遵循数据可视化规范：绿=机会/强势、红=风险/弱势、黄=中性/警戒，浅底深字保证可读性；热力图用同一色系的深浅表达强弱，避免红绿对色盲不友好的问题（辅以图标/文字标签）。

# 八、每日自动运行方案

-   **运行时点：**在每个美股交易日收盘后运行（美东约 16:30
    之后，即太平洋时间约 13:30 之后）。用定时任务（scheduled
    task）触发，而非本地脚本，确保稳定。

-   **每日流程：**拉取最新数据 → 计算评分与制度 → 与昨日快照对比 →
    生成当日仪表盘与提示 → 推送给你。

-   **增量对比：**重点是「变化」------制度是否切换、新增了哪些风险/机会提示，报告顶部优先呈现这些增量，而非每天重复全量。

-   **健壮性：**非交易日（周末/假期）自动跳过；数据缺失或异常时发出「数据告警」而非给出错误结论。

+-----------------------------------------------------------------------+
| **落地建议**                                                          |
|                                                                       |
| 先手动跑通、验证几周信号靠谱后，再挂成每日自动任务。自动化最好同      |
| 时保留「原始数据+当日快照」，方便日后回看某天的判断依据、以及做回测。 |
+=======================================================================+
+-----------------------------------------------------------------------+

# 九、回测与验证方法

+-----------------------------------------------------------------------+
| **验证的目标不是收益，而是「风险提示是否及时」**                      |
|                                                                       |
| 趋势系统必然滞后，它抓不住精确                                        |
| 顶底。要验证的是：在历史上的大跌来临前或初期，系统能否把制度切到防御  |
| 、发出风险提示，从而帮你避开「大而久」的下跌------而不是它能赚多少。  |
+=======================================================================+
+-----------------------------------------------------------------------+

-   **方法：**回放历史数据，标注每一次制度切换的日期，对比其后大盘的实际走势（回撤/反弹）。

-   **关键案例：**2008 金融危机、2020 疫情崩盘、2022 加息熊市、2018 Q4
    急跌------看系统在这些节点是否及时转防御。

-   **评估指标：**① 提前/滞后天数；② 转防御后规避的回撤幅度；③
    假警报（whipsaw）次数与代价。

-   **预期管理：**坦诚承认：V
    型底系统会反应慢、震荡市会有假信号；它的价值在于避免深度熊市，而非抓住每次波动。

# 十、局限、防过拟合与免责

  ----------------------------------------------------------------------------
  **局限**                            **应对**
  ----------------------------------- ----------------------------------------
  滞后性：信号出现在顶/底附近之后     接受它------目标是避大跌，不是抄顶摸底

  震荡市假信号（whipsaw）             多指标共振 + 制度确认机制降低频率

  参数敏感/过拟合风险                 一律用行业标准参数，不做历史最优化

  依赖数据质量                        数据异常时告警而非硬给结论

  不含基本面与黑天鹅                  定位为风险提示器，需与其他信息结合
  ----------------------------------------------------------------------------
